Opdracht 1: Voor block en inline block worden een height width gegenereerd. Voor Inline niet. Voor inline worden de waarden als woorden op een pagina gezet

Opdracht 2: Als de div tags onder elkaar staan in html ontstaat er een witruimte. Als je deze allemaal aan elkaar zou plakken op dezelfde regel is er geen witruimte.

Opdracht 3: Er is text-align: center gebruikt bij het div element daardoor is het zo dat de tekst horizontaal staat. Wat betreft vertical-align, deze eigenschap is van toepassing op inline of inline-block elementen en beïnvloedt de verticale uitlijning van de inhoud binnen het element, maar heeft geen effect op block-level elementen zoals div. Daarom heeft vertical-align: center; geen effect op de tekst in de div-elementen. Om tekst verticaal te centreren, kun je inderdaad de line-height eigenschap gebruiken, vooral als je slechts één regel tekst hebt. Bv: line-height: 100px; Voor meerdere regels tekst wordt vaak de tabel-celtruc gebruikt, waarbij de display: table-cell; eigenschap wordt gebruikt om verticale centrering te bereiken.In je 'styles.css' bestaat er echter maar één regel voor class="area tall", en deze regel wordt toegepast op elementen met deze klasse. Als er meerdere regels zijn die de hoogte instellen voor hetzelfde element, zal de regel die als laatste wordt gedefinieerd in de CSS van toepassing zijn. Dit wordt bepaald door de cascading order (overerving) in CSS.
